      스타일 가이드 훑어보기

      문서를 위한 스타일 가이드

      ‘스타일 가이드'라는 단어를 들으면 무엇이 생각나나요? IT 회사에서 근무하거나, 개발과 관련된 직군이라면, ‘스타일 가이드'라는 단어를 듣고 ‘코드를 작성할 때 코드에 적용해야 하는 스타일을 정리한 스타일 가이드’를 생각할 것입니다. 스타일 가이드가 있으면 프로젝트에 참여한 인원의 수에 상관없이 일관된 스타일의 코드를 생산할 수 있습니다. 스타일 가이드가 없어도, 개발을 하는 데는 문제가 없습니다. 하지만 스타일 가이드가 있다면 코드 전체가 일관된 스타일을 유지하는 데 도움이 됩니다. 그래서 코드를 읽고 파악하는 시간을 줄여주니, 커뮤니케이션 비용을 줄여주는 효과가 있습니다. 그런데 스타일 가이드는 코드뿐만이 아니라 문서에도 필요합니다.

      개발과 관련된 일을 하다 보면, 다양한 종류의 문서를 작성하게 됩니다. 예를 들자면, API를 설명하는 API 문서(API Doc)가 있습니다. 사내에서 자체적으로 개발해서 사용하는 사내 기술 제품이 있다면, 해당 기술 제품 사용법을 설명하는 매뉴얼 문서가 있을 것입니다. 서버나 클라이언트 앱을 배포하는 배포 방법을 설명하는 배포 정책 문서도 있습니다. 라이브러리를 설치하고, 사용하는 방법을 설명하는 문서는 어떨까요? 라이브러리를 사용하는데 필요한 커맨드, 라이브러리의 코드를 설명하기 위해서 많은 양의 코드 예제를 포함하고 있을 것입니다. 이렇듯, 개발자들은 개발과 관련된 다양한 종류의 문서를 작성하고 읽게 됩니다. 그런데 만약 문서를 작성하는 사람이 제각기 자기만의 스타일대로 문서들을 작성한다면 어떨까요? 그래서 각 문서가 제각기 다른 형식을 가지고 있다면 어떨까요? 독자가 문서를 읽어서 정보를 파악하는데 시간이 오래 걸릴 것입니다. 그러니 다소 비효율적이라고 할 수 있겠군요. 그래서 문서도 스타일 가이드가 필요합니다.

      Write the Docs은 스타일 가이드란 ‘문서 전체에서 일관된 스타일 및 어조를 유지하는 데 도움이 되는 일련의 표준’이라고 정의합니다. 그러면 문서 스타일 가이드가 없는 회사의 문서들은 어떨까요? 문서의 양은 방대한데, 각 문서마다 제각각 다른 스타일과 형식을 띄고 있을 겁니다. 그리고 문서의 종류도 다양하겠군요. 제가 근무하고 있는 회사는 API 문서에 적용하는 API 스타일 가이드가 있습니다. 하지만 문서의 종류와 상관없이 전체적으로 적용할 수 있는 공통 스타일 가이드는 아직 없습니다. 그래서 이번 기회에 공통 문서 스타일 가이드를 만들기로 했습니다.

      이 시리즈에서는 제가 여러 IT 회사에서 공개한 좋은 스타일 가이드를 알아보는 리서치 과정을 기록하고 정리했습니다. 스타일 가이드 작성을 시작하기 전에, 각 회사의 스타일 가이드를 훑어보고, 공통점과 차이점에 대해서 알아보려 합니다.

      바로 내용에 들어가기에 앞서, IT 기술과 관련된 정보를 제공하는 문서를 기술 문서, 그 외의 문서는 문서라고 지칭하겠습니다. Microsoft, GitLab 등 많은 회사에서 일반 문서(General documentation)과 기술 문서(Technical documentation)으로 종류를 나누어 각기 다른 스타일 가이드를 제공하고 있기 때문입니다.

      문서 스타일 가이드의 종류

      저는 총 7개 IT 회사의 스타일 가이드를 훑어보았습니다. 그러면서 스타일 가이드도 종류별로 분류할 필요성이 있다고 느꼈습니다. 일단, 문서 스타일 가이드(Documentation style guide), 콘텐츠 스타일 가이드(Content style guide), 계획 가이드(Content planning) 이렇게 총 3 가지 종류로 분류했습니다. 첫 번째, 문서 스타일 가이드는 일반적인 문서를 작성할 때 문서의 형식에 적용할 수 있는 스타일 가이드입니다. 숫자, 날짜, 년도 표기 등등 아주 세세하지만 모호한 부분부터 크게는 문법, 구두점 사용 등등 까지 정의합니다. 두 번째는 콘텐츠 스타일 가이드입니다. 콘텐츠 스타일 가이드는 문서를 작성할 때 지켜야 할 어조(Voice and tone), 문서에서 다루기엔 민감하거나 피해야하는 주제, 문서에 내용을 다루는 방법 등등, 문서의 형식보다 내용에 초점을 맞춘 스타일 가이드입니다. 마지막은 계획 가이드입니다. 문서의 주제를 선정하고, 문서를 구조를 설계하는 등 문서를 작성하기 전에 따르면 좋은 단계를 설명함으로써 독자가 더 좋은 문서를 계획하고 생산할 수 있도록 도와주는 스타일 가이드입니다.

      대분류영문 표기설명
      문서 스타일 가이드Documentation style guide문서를 작성할 때, 문서의 형식에 적용할 수 있는 스타일 가이드.
      콘텐츠 스타일 가이드Content style guide문서의 어조, 문서의 주제 선정, 문서에서 내용을 다루는 방법 등등 문서의 내용에 초점을 맞춘 스타일 가이드.
      계획 가이드Content planning문서를 작성하기 전에 거쳐야 하는 단계를 설명하고 도와주는 스타일 가이드.

      이 3가지 분류도 다시 세세한 분류가 필요했습니다. 예를 들어, Microsoft에서 제공하는 문서 스타일 가이드는 문서를 일반 문서(General Documentation)과 기술 문서(Developer contents)로 크게 2가지로 나누고, 기술 문서는 일반 문서 스타일 가이드를 적용하면서 추가로 기술 문서 스타일 가이드(Developer content style guide)를 적용하도록 권유하고 있습니다. 기술 문서는 다시 참조 문서(Reference documentation), 코드 예제(Code examples)로 분류합니다. 각 문서의 종류에 따라 코드 예시를 작성하는 방법과 기술 용어를 표기하는 방법까지 가이드를 제공합니다. 그래서 앞서 3 가지로 분류했던 스타일 가이드를 다음과 같이 총 8개로 분류했습니다.

      대분류중분류영문 표기설명
      문서 스타일 가이드일반 문서 스타일 가이드General documentation style guide일반 문서 작성 스타일 가이드
      기술 문서 스타일 가이드Technical documentation style guide기술과 관련된 정보를 설명하는 문서를 작성할 때 적용하는 스타일 가이드
      블로그 스타일 가이드Blog style guide회사 블로그에 발행할 문서를 작성할 때 적용하는 스타일 가이드
      콘텐츠 스타일 가이드기술 문서 콘텐츠 스타일 가이드Technical content style guide, Developer content style guide여러가지 종류의 기술 문서를 작성할 때, 기술 주제 선정, 기술 정보를 설명하는 방법 등등 내용에 초점을 맞춘 스타일 가이드
      블로그 콘텐츠 스타일 가이드blog content style guide회사 블로그에 발행할 문서를 작성할 때, 문서의 토픽 설정, 문서의 내용 등에 초점을 맞춘 스타일 가이드
      계획 가이드일반 문서 계획 가이드General documentation planning guide일반 문서를 계획할 때 적용하는 스타일 가이드
      기술 문서 계획 가이드Technical documentation planning guide기술 문서를 계획할 때 적용하는 스타일 가이드
      콘텐츠 계획 가이드Content planning guide문서의 내용에 해당하는 콘텐츠를 계획할 때 적용하는 스타일 가이드

      회사별 스타일 가이드의 특징

      각 회사의 스타일 가이드가 가지고 있는 가장 큰 특징을 간략하게 알아봅니다.

      GitLab

      GitLab은 깃 저장소 및 CI/CD, 이슈 추적, 보안성 테스트 등의 기능이 있는 데브옵스 플랫폼입니다. 다른 회사의 스타일 가이드와 다른 점은 마크다운(markdown) 작성에 대한 가이드를 제공한다는 점입니다. 마크다운에서 페이지를 구성하는 구성 요소 작성법, 마크다운 테스트 하는 방법, 마크다운 린트하는 방법 등등 마크다운과 관련된 자세한 스타일 가이드를 제공합니다.

      IBM

      IBM은 블로그에 글을 작성할 때 따라야 할 IBM blogging style guide를 제공합니다. IBM blogging style guide는 글을 작성할 때 ‘템플릿'을 사용해서 작성하라고 권유하면서, 블로그에 사용하는 템플릿을 2 가지로 분류합니다. 첫 번째는 사용자 스토리(User stories), 두 번째는 정보 블로그(informational blog)입니다.

      블로그 템플릿 종류설명
      사용자 스토리(User stories)블로그 사용자가 프로젝트 경험과 배경에 대해서 제공할 때 사용하는 유형.
      정보 블로그(Information blog)새 릴리즈의 콘텐츠를 설명, 새로운 개념을 설명 또는 특정 웨비나 녹화를 시청해야 하는 이유를 설명할 때 사용하는 유형.

      IBM blogging style은 독자가 문서를 쓸 때 도움이 될 수 있도록, 블로그 템플릿마다 잘 작성된 여러개의 샘플 문서를 제공합니다. 또한 QnA 섹션이 있어 블로그에 올릴 문서를 작성할 때 생길 수 있는 궁금한 사항에 대해 미리 안내하고 있습니다.

      Mailchimp

      Mailchimp는 이메일 기반 마케팅 자동화 플랫폼이자 이메일 마케팅 서비스 회사입니다. Mailchimp를 사용하는 고객들은 뉴스 레터, 광고, 프로모션 정보 등의 콘텐츠를 작성한 뒤 Mailchimp를 사용해 이메일로 발송하고, Mailchimp는 이러한 이메일 마케팅 활동을 기술적으로 지원합니다. Mailchimp는 이메일 마케팅 서비스를 이용하는 고객들이 더 좋은 성과를 낼 수 있도록, 다음과 같이 총 6가지나 되는 다양한 종류의 콘텐츠 작성을 도와주는 콘텐츠 스타일 가이드를 제공합니다.

      #가이드 영문 이름설명
      1How to write Educational Content교육 자료 가이드
      2Writing Legal content법문서 가이드
      3Writing Email Newsletters이메일 뉴스레터 가이드
      4Writing for Social Media소셜 미디어 가이드
      5Writing for Accessibility접근성 가이드
      6Writing for Translation번역 가이드

      Mailchimp의 콘텐츠 스타일 가이드는 콘텐츠를 계획하는데 도움이 되는 콘텐츠 계획 가이드도 있습니다. Creating Structured Content 섹션에서는 콘텐츠를 크게 2가지 종류로 나눕니다. 하나는 구조화된 콘텐츠, 다른 하나는 구조화되지 않은 콘텐츠입니다. 구조화된 콘텐츠의 예로는 기술과 교육 문서를 들고 있으며, Creating Structured Content가 권장하는 가이드라인을 적용할 수 있다고 설명합니다.

      Mailchimp의 콘텐츠 스타일 가이드는 독특하게도 콘텐츠 스타일가이드의 독자를 Mailchiimp에 근무하는 직원, 다른 단체에 근무하는 직원 이렇게 총 2가지로 나눈다는 걸 스타일 가이드 첫 장에 명시하고 있습니다.

      MDN

      MDN은 웹 표준, 모질라 프로젝트에 대한 개발 문서가 있는 모질라 공식 웹사이트입니다. 다른 회사의 스타일 가이드와 다른 점은 'SEO를 고려한 글쓰기' 섹션이 있다는 점입니다.

      Microsoft

      일반 문서 스타일 가이드, 기술 문서 스타일 가이드, 콘텐츠 계획 가이드 등등 PDF로 총 1,000장에 달하는 분량의 스타일 가이드를 언어 별로 제공합니다.

      Microsoft에서 제공하는 문서 스타일 가이드는 문서를 일반 문서(General documentation)와 기술 문서(Developer contents)로 크게 2가지로 나누고, 기술 문서는 일반 문서 스타일 가이드의 규칙을 적용하면서 추가로 기술 문서 스타일 가이드(Developer contents style guide)를 적용하도록 권유하고 있습니다. Microsoft의 기술 문서 스타일은 기술 문서를 참조 문서(Reference document), 코드 예제(Code examples) 2가지 종류로 나눕니다.

      기술 문서 종류설명
      참조 문서(Reference document)클래스, 라이브러리, 모델 및 프로그래밍 언어 구성을 포함해 기술 및 언어와 관련된 프로그래밍 요소에 대한 세부 정보를 제공.
      코드 예제(Code examples)프로그래밍 요소를 사용해 특정 기능을 구현하는 방법을 보여줌.
